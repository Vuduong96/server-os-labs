# Ubuntu Server Basics

## Why This Lab Matters

Before you can configure DHCP, DNS, or any server service, you need to be comfortable managing a Linux server. This lab covers the core skills every server administrator uses daily.

## What You Will Learn

| Skill | Why It Matters |
|---|---|
| User & group management | Control who can access what |
| File permissions | Protect sensitive config files |
| systemctl | Start, stop, enable services |
| Disk & memory monitoring | Know when your server is running out of resources |
| SSH configuration | Secure remote access to your server |

## The Linux Server Mindset

On a Linux server there is **no GUI** — everything is done through the terminal. This is actually an advantage:

- Faster to automate
- Less memory used
- Easier to manage remotely
- Scripts can repeat tasks perfectly

## Lab Environment

- **OS:** Ubuntu Server (latest LTS)
- **User:** root (you have full admin access)
- **Time:** approximately 45 minutes

> **Tip:** If you get stuck at any step, read the error message carefully — Linux error messages are usually very descriptive about what went wrong.
